# Profit Analytics Platform

Веб-интерфейс дашборда для малого и среднего бизнеса: маркетинговые расходы, выручка, закуп (COGS), розничные цены по SKU, юнит-экономика по товарам и в целом по бизнесу, каналы и текстовые AI-инсайты (демо).

## Возможности

- Выбор **месяца** и **года** с отображением периода в шапке.
- Сводка «общая картина» с раскрывающимися блоками и KPI с динамикой к прошлому месяцу (зелёный / красный).
- Таблицы каналов и SKU с раскрывающимися пояснениями.
- Статическая вёрстка на **HTML + Tailwind CDN**, логика на **Vanilla JS** (ES-модули).

## Структура репозитория

```
├── index.html          # точка входа Vite и страница приложения
├── src/
│   ├── main.js         # рендер дашборда, расчёты по периоду
│   └── data/
│       └── periods.js  # демо-данные по месяцам (замените на API)
├── package.json
├── vite.config.js
├── LICENSE
└── README.md
```

## Требования

- [Node.js](https://nodejs.org/) **18+**

## Быстрый старт

```bash
npm install
npm run dev
```

Откроется локальный сервер (по умолчанию [http://localhost:5173](http://localhost:5173)) с hot reload.

## Сборка для продакшена

```bash
npm run build
```

Артефакты появятся в каталоге `dist/`. Раздавайте их любым статическим хостингом (Nginx, S3, Cloudflare Pages, Netlify и т.д.).

Просмотр собранной версии локально:

```bash
npm run preview
```

## Открытие без Node

Файл `index.html` можно открыть в браузере напрямую, но из-за ES-модулей (`import`) страница корректно загрузит скрипты только с **HTTP-сервера** (как при `npm run dev` или после деплоя `dist`). Двойной клик по `index.html` с `file://` в части браузеров даст ошибку CORS для модулей.

## Дальнейшее развитие

- Подключить реальные графики (например, Chart.js) вместо заглушек.
- Заменить `src/data/periods.js` на запросы к вашему API.
- Вынести токены Tailwind в PostCSS-сборку, если понадобится offline и контроль purge.
